## Setup

Nestia uses a Go-backed native transform hosted by `ttsc`. Configure it
directly in `tsconfig.json`; the old interactive `npx nestia setup` wizard is
not used.

## Install

```bash copy
npm install --save @nestia/core @nestia/fetcher typia
npm install --save-dev nestia @nestia/sdk ttsc@^0.10.2 typescript@~6.0.3
```

Go 1.26 or newer must be available on `PATH`, or through `TTSC_GO_BINARY`, while
prebuilt native packages are not distributed.

Install `@nestia/sdk` as a runtime dependency instead of a dev dependency when
the application imports `NestiaSwaggerComposer` to serve Swagger from the app
process.

## TypeScript

For controller validation and serialization, add the core native transform.

```json filename="tsconfig.json" showLineNumbers copy
{
  "compilerOptions": {
    "strict": true,
    "plugins": [
      { "transform": "typia/lib/transform", "enabled": false },
      { "transform": "@nestia/core/native/transform.cjs" }
    ]
  }
}
```

For SDK, Swagger, runtime Swagger composer, and e2e generation, include the SDK
descriptor before the core native descriptor.

```json filename="tsconfig.json" showLineNumbers copy
{
  "compilerOptions": {
    "strict": true,
    "plugins": [
      { "transform": "typia/lib/transform", "enabled": false },
      { "transform": "@nestia/sdk/lib/transform" },
      { "transform": "@nestia/core/native/transform.cjs" }
    ]
  }
}
```

The `typia` entry is intentionally disabled. The Go binary runs the typia
transform internally, but keeping this entry lets `ttsc` and related tools see
the host plugin configuration.

## Scripts

Use `ttsc` instead of `tsc` for transformed builds.

```json filename="package.json" showLineNumbers copy
{
  "scripts": {
    "build": "ttsc",
    "start:dev": "ttsx src/main.ts"
  }
}
```

The `nestia` CLI commands for generation read `nestia.config.ts` and the project
`tsconfig.json`.

```bash copy
npx nestia init
npx nestia sdk
npx nestia swagger
npx nestia e2e
npx nestia all
```

`nestia` is a thin dispatcher. `start` and `template` can bootstrap a project
from the CLI package itself, while `dependencies`, `init`, `sdk`, `swagger`,
`e2e`, and `all` run the project-local `@nestia/sdk` command with your local
`ttsc` and TypeScript versions.

## Boilerplate

Use `start` for the full NestJS starter project and `template` for the smaller
SDK template.

```bash copy
npx nestia start my-app
npx nestia template my-sdk
```

If compilation fails, run `npx ttsc -p tsconfig.json` directly to see the native
`ttsc-nestia` diagnostic.
