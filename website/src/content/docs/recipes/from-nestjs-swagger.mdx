# Migrating from `@nestjs/swagger`

Already using `@nestjs/swagger` and want to drop the decorators without losing your documentation? Lift the strings into JSDoc and the constraints into `tags.*` intersections.

You can migrate file by file — Nestia ignores `@ApiProperty()` while it is still there, so no big-bang switch is needed.

---

## Class → interface

Before:

```ts
import { ApiProperty } from "@nestjs/swagger";

export class CreateArticle {
  @ApiProperty({ description: "Article title", minLength: 3, maxLength: 50 })
  title!: string;

  @ApiProperty({ description: "Article body" })
  body!: string;
}
```

After:

```ts
import { tags } from "typia";

export interface ICreateArticle {
  /**
   * Article title
   */
  title: string & tags.MinLength<3> & tags.MaxLength<50>;

  /**
   * Article body
   */
  body: string;
}
```

The class becomes an interface (or stays a class — Nestia doesn't care). `description` moves into JSDoc, constraints move into intersection tags.

## Property options mapping

| `@ApiProperty()` option | Nestia equivalent                                       |
| ----------------------- | ------------------------------------------------------- |
| `description: "..."`    | JSDoc summary on the property                           |
| `title: "..."`          | JSDoc `@title ...`                                      |
| `example: 42`           | JSDoc `@example 42`                                     |
| `default: "x"`          | JSDoc `@default x`                                      |
| `type: "string"`        | Already implied by the TypeScript type                  |
| `format: "uuid"`        | `& tags.Format<"uuid">`                                 |
| `format: "email"`       | `& tags.Format<"email">`                                |
| `format: "date-time"`   | `& tags.Format<"date-time">`                            |
| `minLength: 3`          | `& tags.MinLength<3>`                                   |
| `maxLength: 50`         | `& tags.MaxLength<50>`                                  |
| `pattern: "^[a-z]+$"`   | `& tags.Pattern<"^[a-z]+$">`                            |
| `minimum: 0`            | `& tags.Minimum<0>`                                     |
| `maximum: 100`          | `& tags.Maximum<100>`                                   |
| `enum: ["a","b"]`       | `"a" \| "b"` (plain TypeScript union)                   |
| `nullable: true`        | `\| null` in the TypeScript type                        |
| `required: false`       | `?` on the property                                     |
| `isArray: true`         | `T[]` in the TypeScript type                            |
| `oneOf: [...]`          | TypeScript union                                        |

## Controller method docs

Before:

```ts
@ApiOperation({ summary: "Create article", description: "Stores a new article." })
@ApiResponse({ status: 201, type: Article })
@ApiResponse({ status: 400, description: "Invalid payload" })
@Post()
async create(@Body() input: CreateArticle): Promise<Article> { ... }
```

After:

```ts
import { TypedBody, TypedException, TypedRoute } from "@nestia/core";

/**
 * Create article
 *
 * Stores a new article.
 */
@TypedException<IBadRequest>(400, "Invalid payload")
@TypedRoute.Post()
async create(@TypedBody() input: ICreateArticle): Promise<IArticle> { ... }
```

The summary becomes the JSDoc first line, the description becomes the JSDoc body, success responses are inferred from the return type, and error responses move to `@TypedException`.

## Security schemes

Before, in `main.ts`:

```ts
const config = new DocumentBuilder()
  .setTitle("My API")
  .addBearerAuth()
  .build();
```

After, in `nestia.config.ts`:

```ts
const config: INestiaConfig = {
  input: () => NestFactory.create(AppModule),
  output: "src/api",
  swagger: {
    output: "dist/swagger.json",
    info: { title: "My API", version: "1.0.0" },
    security: {
      bearer: { type: "apiKey", name: "Authorization", in: "header" },
    },
  },
};
```

---

## Running both during the transition

While migrating, leave `SwaggerModule.setup` wired up if you like. When you're ready to switch fully, replace it with `NestiaSwaggerComposer.document` from `@nestia/sdk` — see **[Swagger → Overview](/docs/swagger)**.

---

## When you're done

Remove `@nestjs/swagger` from `package.json` and grep your codebase for `ApiProperty` to confirm nothing is left behind. The remaining JSDoc / type annotations are all the documentation you need.
