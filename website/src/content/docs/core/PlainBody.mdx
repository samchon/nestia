## Outline
```typescript filename="@nestia/core" showLineNumbers
export function PlainBody(): ParameterDecorator;
```

Decorator that reads the **raw `text/plain`** request body as a string.

`@PlainBody()` exists because `@Body()` (and `@TypedBody()`) always try to parse the body as JSON. When the client sends `Content-Type: text/plain` — markdown payloads, memos, copied-pasted logs, raw template content — both decorators throw or produce `{}`. `@PlainBody()` short-circuits the parser and hands you the body as `string`.

<Callout type="info">

For other content types use the typed variant that matches:

  - `application/json`: [`@TypedBody()`](./TypedBody)
  - `application/x-www-form-urlencoded`: [`@TypedQuery.Body()`](./TypedQuery#typedquerybody)
  - `multipart/form-data`: [`@TypedFormData.Body()`](./TypedFormData#TypedFormDatabody)

</Callout>

## How to use

```typescript copy filename="MemoController.ts" showLineNumbers
import { PlainBody, TypedRoute } from "@nestia/core";
import { Controller } from "@nestjs/common";

@Controller("memos")
export class MemoController {
  @TypedRoute.Post()
  public store(@PlainBody() body: string): { stored: number } {
    return { stored: body.length };
  }
}
```

Send the request with `Content-Type: text/plain`:

```bash filename="Terminal"
curl -X POST http://localhost:3000/memos \
  -H "Content-Type: text/plain" \
  --data-binary @memo.md
```

The handler receives the file's contents verbatim, including newlines, trailing whitespace, and any non-ASCII characters in UTF-8.

## Behavior

- The decorator only accepts requests whose `Content-Type` resolves to `text/plain` (with optional `charset=...` parameters). Any other content type triggers `400 Bad Request` with the message `Request body type is not "text/plain".`
- When the body is empty (no `Content-Length` header or zero bytes), the parameter is `undefined`.
- No validation is run on the body. If you need to enforce a shape on the string, use TypeScript tags or a manual guard inside your handler:

```typescript
import type { tags } from "typia";

@TypedRoute.Post()
public store(@PlainBody() body: string & tags.MinLength<1>): void;
```

<Callout type="warning">

`@PlainBody()` is incompatible with `@TypedBody()` on the same handler — only one request-body decorator can run per method.

</Callout>

## When not to use

If your endpoint accepts JSON, use [`@TypedBody()`](./TypedBody). If it accepts form-encoded payloads, use [`@TypedQuery.Body()`](./TypedQuery#typedquerybody) or [`@TypedFormData.Body()`](./TypedFormData#TypedFormDatabody). `@PlainBody()` is the right choice only when the contract really is "give me the body as a string."
